Apache Calcite + Graal / Truffle でクエリエンジンを作ってる話

@Masayuki Takahashi · Dec 19, 2020 · 3 min read

Motivation

MPP クエリエンジンに興味があったのですが、仕事で Impala を使ってみて益々興味が湧き、自分で作ってみたいと思うようになって作り始めました。 イチから作るのはとても時間がかかるので、色んなライブラリを使っています。今回はそれらの紹介をしたいと思います。

How Query Engines Work

最初はどうやってクエリエンジンを作れば良いのか分からなかったので、とりあえず参考書等を探して入り口を探しました。Amazon で書籍を探すといくつか洋書が出てくるのですが、それを読んでも実装できる気がしなかったので、まずは Apache Drill のコードを読んでいました。

最近は、Apache Arrow の PMC の Andy Groove が『How Query Engines Work』という書籍をリリースしているので、興味のある方は読んでみてください。彼自身も Rust と Apache Arrow を使って DataFusion というクエリエンジンを書いています。

Parsing SQL and Building an execution plan

SQL の parse から実行計画(Logical Plan)の構築までは、Apache Calcite を使っています。Apache Calcite については以前 blog を書いたので、そちらを参考にしてください。

https://masayuki038.github.io/blog/2019/06/04/calcite-code-reading-part1/

あとは、Logical Plan から Physical Plan を生成し、Physical Plan から先で SQL の処理を行い、結果を返すだけです。

Physical Plan の生成も、Apache Calcite がサポートします。Physical Plan の RelNode のクラスを作り、Logical Plan の各 RelNode を Physical Plan の RelNode に変換していきます。1:1 で対応させていく必要はなく、2 つの RelNode を 1 つに纏めたり、RelNode を組み替えたりすることができます。

Data Source

内部では Apache Arrow 形式のデータを処理しています。Apache Arrow については、以下を参照してください。

https://www.slideshare.net/kou/ossforumapachearrow

現状は Arrow 形式のデータを予めカラムごとに分けて出力し、実行計画を見て必要なカラムのファイルを読むようにしています。

Code Generation

Apache Calcite の標準実装的な位置付けである Enumerable Adapter は、実行計画を参照しながら Java のコードを生成し、最後にそれらをコンパイルして実行し、結果を返すようになっています。Apache Calcite の arrow adapter を書いていた頃は、Enumerable Adapter と同じように Java のコードを生成していたのですが、Java のコードを生成する Java コードを書くのがどうにもシンドくなってしまい、別の方法を探すことにしました。

暫くして Graal / Truffle を知り、コード生成の部分に Truffle が使えないかと思うようになりました。そして Web を探してみた結果、Calcite + Truffle で実装しているプロジェクトを見つけました。

https://github.com/fivetran/truffle-sql

このプロジェクトの開発はすでに終わっているようですが、このプロジェクトを fork して Arrow Adapter のコードを組み合わせて動かしてみるようにしたのが、trffle-arrow のベースになっています。

Expression Evaluation

Apache Calcite では、実行計画の Scan / Filter / Project 等を RelNode、SQL 文の中の式等を RexNode というオブジェクトとして扱います。

truffle-sql では、この RexNode を Truffle の Node に置き換えていきます。そして RelNode を下から順に実行していきます。RelNode の中では Truffle の Node を評価し、結果を Truffle の VirtualFrame に書き込みます。最後に VirtualFrame から結果を取り出して、JDBC の ResultSet 経由で値を取り出せるようにしています。

JDBC API

Apache Calcite の adapter の実行結果は JDBC API 経由で取得できるようになっています。しかし、ネットワークを介して結果を取得することはできません。そこで、Apache Avatica を使ってネットワークを介して結果を取得できるようにしています。

Source Code

Scan / Filter / Project / Aggregate まで実装したものを以下に置いています。

Conclusion

クエリエンジンを作るために使っているライブラリを紹介しました。Java 系の言語を使って SQL クエリエンジンを作るのであれば、その取っ掛かりとして Apache Calcite を見てみることをお勧めします。
